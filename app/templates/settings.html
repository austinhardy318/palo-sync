DeepDiff Path Syntax

DeepDiff paths describe the location of values in your configuration structure. They start with root and use bracket notation to navigate through nested dictionaries.

Exact Path Example:

root['devices']['entry']['deviceconfig']['system']['hostname']['_text']

This path targets the hostname text value in the device configuration system settings.

How to Find Paths:

  1. Run a diff on your configurations
  2. Click "View Raw Diff" to see the DeepDiff output
  3. Look for paths like root['section']['key'] in the differences
  4. Copy the exact path to ignore that field

Regex Path Example:

.*\['_last_modified'\]

This regex matches any path containing ['_last_modified'], useful for ignoring timestamp fields across the configuration.

Significant Digits for Numeric Comparisons

Controls how many significant digits to consider when comparing numeric values. Small rounding differences won't be flagged as changes.

  • Blank/0: Exact numeric comparison (default)
  • 3-5: Useful for ignoring minor floating-point precision differences
  • 5-7: Useful for timestamps or measurements with small precision variations

Example:

With significant_digits=5, values like 99.99987 and 99.99992 would be considered the same (both round to 5 significant digits: 99.999), while 99.99 and 100.01 would still be different.
